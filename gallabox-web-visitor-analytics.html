Improve Analytics — 3 events missing from the widget script
Add these to _gb.js to unlock the full engagement picture and enable data-driven auto-open decisions
Engineering action
👁
widget_visible — widget enters the viewport
Currently "Widget Visible" is estimated from page load. Visitors on long pages who never scroll to the widget are counted as having seen it. Fire this event on IntersectionObserver entry.
Unlocks: true impression rate · above-vs-below-fold split · scroll depth at widget appearance
GB.track('widget_visible', { agent_id, scroll_pct, time_on_page_s })
⚡
open_type — manual click vs auto-open vs proactive message
Without knowing how the widget was opened, the "opened → chat" rate can't be split by trigger type. This split is the core input for evaluating whether to enable auto-open and for A/B testing trigger rules.
Unlocks: auto-open vs manual conversion rate · proactive message effectiveness · data-driven auto-open on/off decision
GB.track('widget_open', { open_type: 'click|auto|proactive', trigger_rule_id })
⏱
session_context — scroll depth + dwell time at first message sent
When a visitor sends their first message, capture how long they spent on the page and how far they scrolled. This reveals the engagement threshold that precedes a chat, directly informing the optimal auto-open delay and scroll trigger.
Unlocks: optimal dwell time for auto-open · high-intent page sections · bounce signal (chat at scroll < 20% means auto-open triggered too early)
GB.track('chat_start', { scroll_pct, dwell_s, page_url, visitor_touch_count })
